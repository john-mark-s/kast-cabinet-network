# CLAUDE.md — Kast Cabinet Network

## Project Overview

Build **Kast Cabinet Network** — an open-source transparency dashboard that maps the financial, corporate, and lobbying connections of Chile's 24 cabinet ministers (inaugurated March 11, 2026). The tool cross-references public data from ChileCompra (procurement), InfoLobby (lobbying meetings), InfoProbidad (asset/interest declarations), and SII (corporate registry) using the **RUT** (Chile's universal tax ID) as the linking key.

### Architecture

Fully serverless. Python scrapers run as GitHub Actions on cron schedules, producing static JSON files. A React (Vite + TypeScript) frontend on GitHub Pages consumes those JSON files. No backend server, no database — just flat files in the repo. Git history is the audit trail and diff engine.

### Three core views

1. **Red de conexiones** — Interactive force-directed graph showing entities and connections with red flag indicators
2. **Cambios recientes** — Weekly changelog showing what's new or changed, auto-generated by diffing graph.json against previous git commit
3. **Fuentes y verificación** — Source directory + step-by-step verification guide + legal basis, so any visitor can independently verify every data point

### Phase transitions

- **Phase 1 (manual seed):** Static graph.json built from config.py + manual research. All 24 ministers listed, Quiroz fully mapped as proof-of-concept.
- **Phase 2 (live scraping):** GitHub Actions cron jobs populate real data; diff engine auto-generates changelog.
- **Phase 3 (expansion):** Add remaining ministers, expand corporate network, link InfoLobby meetings to ChileCompra contracts.

## Data Architecture

### The RUT as linking key

Every entity in Chile — person or company — has a RUT. The scraping pipeline resolves RUTs from:
1. **config.py** — manually researched seed data
2. **InfoProbidad declarations** — ministers declare company participations with RUTs
3. **ChileCompra contracts** — each order links buyer (institution RUT) ↔ supplier (company RUT)
4. **SII bulk file** — confirms entity name, activity, address for any RUT

### Graph data model

```
Node types: person | company | institution | corporate_client
Edge types: owned | heads | family | works_at | contract | advisory | met_with | received_gift | board | govt_role | rumored
Red flags: SELF_DEALING | REVOLVING_DOOR | DIRECT_AWARD_CLUSTER | LOBBY_TO_CONTRACT | CONCENTRATION | CORPORATE_RESTRUCTURING | GOV_CONTRACTOR | COLLUSION_LINK | ACTIVE_CONTRACT
```

### File layout

```
data/
  raw/                    # gitignored — raw API responses
  processed/
    chilecompra/          # {rut_normalized}.json per company
    infolobby/            # {minister_id}.json per minister
    infoprobidad/         # {minister_id}.json + discovered_ruts.json
    sii/                  # companies.json bulk lookup
  graph/
    graph.json            # full network (nodes + edges + meta)
    ministers/            # {minister_id}.json per-minister detail
    meta.json             # last-updated timestamps per source
    changelog.json        # weekly diff feed

frontend/public/data/     # copy of data/graph/ for the static site
```

## Scraping Scripts

### scripts/utils.py
- `normalize_rut(rut)` → strips dots, ensures `12345678-9` format
- `format_rut_for_api(rut)` → adds dots back for ChileCompra: `12.345.678-9`
- `validate_rut(rut)` → módulo 11 check digit
- `api_get(url, params, retries=3)` → tenacity retry wrapper

### scripts/config.py
Master seed data: all 24 ministers with known companies, associates, government contracts, collusion links, and board memberships.

### scripts/scrape_chilecompra.py
Queries ChileCompra REST API for each company RUT. Saves to `data/processed/chilecompra/{rut_normalized}.json`.

### scripts/scrape_infolobby.py
Queries InfoLobby REST API by minister name. Handles pagination. Saves to `data/processed/infolobby/{minister_id}.json`.

### scripts/scrape_infoprobidad.py
Searches InfoProbidad open data by minister name. Extracts company RUTs. Appends to `discovered_ruts.json`.

### scripts/scrape_sii.py
Downloads SII bulk file. Looks up all known RUTs. Flags same-address entities.

### scripts/build_graph.py
Combines config.py + all processed data → `data/graph/graph.json`. Detects red flags algorithmically.

### scripts/diff_graph.py
Compares current graph.json vs `git show HEAD~1:data/graph/graph.json`. Outputs `data/graph/changelog.json`.

## Red Flag Definitions

| Flag | Trigger | Severity |
|------|---------|---------|
| SELF_DEALING | Minister heads ministry that awarded contract to their own company | Critical |
| REVOLVING_DOOR | Person moved between minister's private firm and a government role within 24 months | High |
| DIRECT_AWARD_CLUSTER | Same supplier received 3+ direct awards (trato directo) from same buyer in 12 months | High |
| LOBBY_TO_CONTRACT | Entity that lobbied a ministry later received a contract from that ministry (within 18 months) | High |
| CONCENTRATION | Single supplier accounts for >40% of ministry spend in a category | Medium |
| CORPORATE_RESTRUCTURING | Company renamed or restructured within 12 months of public scrutiny | Medium |
| GOV_CONTRACTOR | Entity with active government contracts | Info |
| COLLUSION_LINK | Entity named in competition authority (TDLC/FNE) proceedings | High |
| ACTIVE_CONTRACT | Contract currently active (no end date or end date in future) | Info |

**Important:** Flags are signals for investigation, not evidence of wrongdoing. Every flagged node includes source links so readers can verify independently.

## Frontend Guidelines

### Design system
- Dark mode: bg `#0F172A`, panels `#1E293B`, borders `#334155`
- Chilean flag accents: red `#CE1126` (alerts/flags), blue `#0039A6` (sparingly)
- Mobile responsive — graph collapses to list view on small screens

### Graph styling
- Nodes: person=blue circle, company=gray square, institution=green hexagon, corporate_client=purple diamond
- Flagged nodes: dashed red border, pulsing animation
- Edges: solid=confirmed, dashed=rumored; color by type

### Localization
- UI language: Spanish
- Data: mix of Spanish and English (source-dependent)
- Dates: ISO 8601 internally, `dd MMM yyyy` display

## Legal & Ethical Framework

### Legal basis
All data used is either publicly available under Chilean transparency laws or published as open data by government portals:
- **Ley 20.285 (2008)** — Transparency law: government must publish data
- **Ley 20.730 (2014)** — Lobby law: all meetings with senior officials must be registered
- **Ley 20.880 (2016)** — Probity law: senior officials must declare all assets and interests

### Methodology disclaimer
> "Este proyecto presenta conexiones documentadas en fuentes públicas. Los patrones identificados son señales para la investigación periodística, no prueba de irregularidades. Cada dato incluye enlaces de verificación. Errores pueden reportarse abriendo un issue en GitHub."

## Development Commands

```bash
# Install Python deps
pip install -r requirements.txt

# Run scrapers (need API keys in .env)
python scripts/scrape_chilecompra.py
python scripts/scrape_infolobby.py
python scripts/scrape_infoprobidad.py
python scripts/scrape_sii.py

# Build graph
python scripts/build_graph.py
python scripts/diff_graph.py

# Copy to frontend
cp -r data/graph/* frontend/public/data/

# Frontend dev
cd frontend && npm install && npm run dev

# Frontend build
cd frontend && npm run build
```
